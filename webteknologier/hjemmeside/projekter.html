  Bongospil  
Hvad handler det om?

I andelning med vores forløb Spil, Leg og Samarbejde, skulle vi lave et eller andet spil, der opfylder tre krav:

  • Kan spilles i af to eller flere personer, der fysisk er i nærheden af hinanden.
  • Involverer et samarbejdende eller konkurrerende element, gerne inspireret af sport.
  • Styres med Makey Makey og ikke direkte fra computeren.

Vi besuttede os derfor, at vi gerne ville lave et simpelt bongo spil, hvor der kommer røde og blå bolde kørende ned, langs en bane. Når disse kugler så kommer hen i et bestemt område skal man slå på enten den venstre bongo (for røde kugler), eller på den højre (for blå kugler). Hver gang man så ramte rigtigt, vil man så få et point.

Dette spil var en forsimplet genskabelse af det japanske spil Taiko No Tatsujin.

Hvordan opfyldte vi kravene?

Kravet om at flere skulle kunne spille sammen:
For at opfylde dette krav, sørgte vi for at der var et pointsystem, i spillet. Man kan derefter sammenligne deres score.

Skal involvere samarbejdende eller konkurrerende element:
Vha. pointsystemet kan spillere både konkurrere med sig selv og med andre, ved at se hvem der kan få den bedste highscore. Hvis man gerne vil samarbejde i spillet, kan man være to om at spille det på samme tid, hvor en har de blå- og den anden har de røde kugler.

Skal styres af Makey Makey:
For at opfylde dette krav, satte vi først inputsne til spillet, til at være venstre- og højre pil. Derefter lavede vi nogle "bongo trommer" ud af to stykke sølvpapir, viklet rundt om to stykker pap. Man kunne derefter koble Makey Makey til komputeren, og inputtet til højre- og venstre pil, til de to plader. Derefter skulle man koble sig selv til ground, også kunne man spille spillet med Makey Makey.

Mulige forbedringer

Her er en liste over, hvordan vi kan forbedre vores spil:

  1. Der skal være to baner, så to folk kan spille på samme tid
  2. Pointsystemet skal forbedres, så man mister point, når man trykker forkert
  3. Kuglerne skal tilpasses til noget musik, så man slår på bongoerne i takt med en bestemt sang
  4. Vi kan ændre spillet, så man skal træde på pladerne, istedet for at slå på dem. Dette vil spille lidt mere ind på sportelementet i vores krav
  5. Der skal være en restart knap, så man ikke skal genindlæse siden
  6. Man skal kunne gemme sin highscore
  7. Lyden virker kun i liver server, så få lyden til at virke på selve hjemmesiden
Forklaring af koden

Hvordan lavede vi cirklerne?

Vi lavede cirklerne vha. JS funktionen constructor().

Med denne funktion kan man pre-definere et bestemt slags objekt, hvilket vil sige, at hver gang man caller funktionen, skal man kun indsætte de parameter som funktionen klader for, også laver den et standarieret objekt med de genstande.

For at få cirklerne til at bevæge sig, lavede vi en funktion move(), der tog vi hver node der eksisterede, og justerede den langs x-aksen med -5px. Dette vil sige, at den bevægede sig mod venstre.

Til sidst lavede vi en funktion display(), der både lavede selve cirklerne, men også farvede den enten rød eller blå. Farven blev valgt ved at JS selv gav hver cirkel værdien 1 eller 2. Hvis værdien er cirklen var 1, vil den være rød, og vice versa.

Man kan derefter calle hvilken som helst af funktionerne man har brug for, ved at skrive new Note(type, x), move() eller display()

Se kode her: 

class Note {
  constructor(type, x) {
    this.type = type; // 1 = red, 2 = blue
    this.x = x; // X position of the note
  }

  move() {
    this.x -= 5; // Adjusts the circles x-position by -5px
  }

  display() {
    if (this.type === 1) {
      fill(255, 0, 0); // Red note
    } else {
      fill(0, 0, 255); // Blue note
    }
    ellipse(this.x, height / 2, 50, 50);
  }
}

Hvordan ved man om man har trykket rigtigt? (Collision detection)

For at se om man rammer rigtigt, lavede vi en funktion der hedder checkHit(). Denne funktion tjekkede om en kugle først er over regristreringsboksen, ved at finde differencen mellem boksens og kuglernes position. Derudover tjekker den også selve værdien af kuglen (om den er rød eller blå), og sammenligner det med spillerens input. Hvis begge disse to konditioner gælder, så fjernes noden, og ens score øges med 1.

Se kode her: 

function checkHit(noteType) {
  for (let i = notes.length - 1; i >= 0; i--) {
    if (abs(notes[i].x - hitZoneX) < 50 && notes[i].type === noteType) {
      score += 100; // Correct hit
      notes.splice(i,1); // Remove hit note
      break;
    }
  }
}

Hvad handler det om?

Efter vores 3.g studieture, og i andledning med vore forløb webteknologier, skal vi nu hjælpe fremtidige RYS-elever få den bedste studietur muligt. Vi skal derfor lave en hjemmeside egent til telefonen, der kan give dem alle nødvændige informationer om destinationen.

Denne hjemmeside havde en række krav den skulle opfylde:

  • En hoveside med en kort introduktion af destinationen.
  • Link og undersider med relevant information
  • Overskrift og afsnit.
  • Billeder eller videoer.
  • Styling med brug af CSS

  PixelPaint  
Hvad handler det om?

I vores forløb billedmanipulation skulle vi lave et billedemanipulationsprogram, hvor man kan manipulere pixelsne på et billede, på pixelniveau. Vores program skulle kunne opfylde en hvis række krav:

  • Kan redigere et billede ved manipulation på pixelniveau
  • Kan tilføje effekter
  • Bestemme hvad der er kerne- og sekundære-faciliteter
  • Udføre en brugertest og en tilretning af projektet

Hvordan opfyldte vi vores krav

Kan redigere et billede på pixledniveau

For at opfylde dette krav, sikrede vi os at vi kunne tegne en streg der fulgte vores mus, og dermed override pixedværdien i en bestemt radius rundt om musets position.

Den nedstående kodestump tager musens tideligere og nuværende position, og tegner en strej imellem dem

Se kode her: 

function mouseDragged(){
  stroke(vRed, vGreen, vBlue)
  line(pmouseX, pmouseY, mouseX, mouseY)
}

Kan tilføje effekter

De effekter vi kunne tilføje, var selve farven og størrelsen af vores strokes, hvilket blev gjordt vha. nogle knapper der linkede til bestemte funktioner, samt en slider, der kan variere stroke tykkelsen

I den nedstående kode, ser vi hvordan farven ændres til f.eks rød

Se kode her: 

function colorRed(){
  vRed = 255;
  vGreen = 0;
  vBlue = 0;
  }

Bestemme kerne- og sekundære faciliteter

Vores primære funktion til dette program var, at den skulle tegne en simpel streg, den streg skulle kunne slettes, og vi skulle gemme billedet

Derudover er vores sekundære funktioner, det med at man kunne ændre farven og stroke size

Udføre en brugertest, og og tilrette vores program

Vi fik lavet en brugertest, hvor vi fik noget dejligt feedback, som at vi skulle have nogle presets til f.eks filtre (som vi egentligt allerede havde planer om at lave), derudover sagde de at det ville være dejligt, hvis der var flere farvemuligheder. Vi havde faktisk i forvejen lavet en HUE-farveselector, men har bare ikke haft muligheden for at implementere den.
